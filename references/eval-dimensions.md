## 9 层维度 × Skill 类型测试重点

| 维度 | 核心检查点 | mcp_based | text_generation | code_execution |
|------|-----------|-----------|----------------|----------------|
| **触发层** | 触发准确性、边界触发 | description 是否清晰区分 TP/TN | 同左 | 同左 |
| **输出层** | 格式合规、内容准确、一致性 | 工具调用参数正确性 | 结构/格式/长度/关键信息 | 输出文件存在 + 内容正确 |
| **业务层** | 规则遵守、IFR=100%、合规 | 每条硬性规则对应 1 条用例 | 规则声明可追溯到 SKILL.md | 命令执行符合规范 |
| **交互层** | 多轮一致性、追问质量 | 缺信息时追问是否精准一次问清 | 同左 | 同左 |
| **健壮层** | 鲁棒性、幻觉检测、安全性 | 异常输入/Prompt注入测试 | 规则声明来源可验证 | 边界输入/异常路径 |
| **效率层** | 响应时间、Token 消耗 | P95 < 15s（S/A级）< 30s（B/C级） | 不适用 | P95 < 15s |
| **设计层** | 复杂度、负向增益 Δ | Δ > 0，否则发布红线 | 同左 | 同左 |
| **工程层** | 覆盖率、环境一致性 | S/A级综合覆盖率 ≥ 85% | 同左 | 同左 |
| **组织层** | 知识时效性 | 规则有有效期标注 | 同左 | 同左 |

## 关键公式

```
IFR（指令遵循率）= 正确遵循硬性规则次数 / 触发硬性规则总次数 × 100%
  → S 级要求 IFR = 100%

一致性得分 = 关键字段完全一致的组数 / 总对比组数 × 100%
  → 目标 ≥ 90%

复杂度得分 = (SKILL.md 行数 / 50) + (模块数 × 2) + (硬性规则数 × 0.5)
  → ≤10 精简  11-20 适中  >20 建议拆分

综合覆盖率 = 功能覆盖率×0.5 + 路径覆盖率×0.3 + 断言覆盖率×0.2
  → S/A级 ≥ 85%，B/C级 ≥ 60%

增益 Δ = Pass Rate(with_skill) - Pass Rate(baseline)
  → Δ > 10% 显著  0<Δ≤10% 偏低  Δ<0 发布红线
```

## 幻觉检测方法

**mcp_based**：验证每个"事实性声明"能在 SKILL.md 或已知业务规则中找到来源。
常见类型：规则幻觉（编造规则）、字段幻觉（编造字段）、数值幻觉（编造金额上限）、流程幻觉（编造审批步骤）。

**text_generation**：对输出中每个"规则性声明"（「根据公司规定...」），在 SKILL.md 搜索来源：
- 找到 → 正常
- 找不到 → `hallucination`，`claim verified = false`

## 防编造可信度参考（断言设计时）

| Skill 类型 | 可信度 | 核心机制 | 断言设计建议 |
|----------|--------|---------|------------|
| mcp_based + 真实执行 | 中（5/10） | transcript 双分离 + claims 交叉验证 | exact_match 断言为主，用 verify_assertions.py 脚本验证 |
| code_execution + 读输出文件 | 高（6.5/10） | 文件系统物理锚点 | 直接断言输出文件内容，不依赖 transcript Return 字段 |
| text_generation | 低（2.5/10） | 全链路 AI 闭环，无客观锚点 | 量化断言（字数/格式）占 ≥50%，发布前须业务方人工确认 |
| mcp_based + MCP 不可用 | 极低（1/10） | 仅规则推断 | **不可用于发布决策，修复 MCP 环境后重测** |
